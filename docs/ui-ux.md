# UI/UX方針

## 1. デザイン原則

- スマホ優先
- ナチュラル / 健康管理アプリ寄り / iOS風
- 家計簿感を出し過ぎず、日常ツールとして軽く使えること

## 2. 主要画面一覧

- ダッシュボード（初期実装優先）
- 献立カレンダー
- 献立リスト
- レシピ管理
- 買い物リスト
- 予算/実績サマリー

## 3. 最初に作る画面

- ダッシュボード

## 4. ダッシュボード表示項目

- 今月予算 / 使用額 / 残額
- 今週予算 / 見込み
- 直近の買い物予定
- 予算超過警告
- 今日の食事枠と対象状態

## 5. カレンダー表示方針

- 月表示を基本
- 食事枠ごとに簡潔表示
- 詳細はタップでドロワー表示（要確認）

## 6. リスト表示方針

- 週次調整しやすい時系列
- 対象外切替と理由記録を1タップ導線化

## 7. 買い物リストUI方針

- 買い物予定日ごとに区切る
- 購入済みチェックを大きく配置
- 価格入力は少操作で完了させる

## 8. 表示項目カスタマイズ方針

- MVP 0: 固定表示（過剰設定を避ける）
- MVP 1: カレンダー/リストの表示項目カスタマイズ

## 9. 並び替えUI方針

- 将来対応: SwapyのようなUIを検討
- MVP 0: 固定レイアウト
