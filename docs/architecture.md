# アーキテクチャ方針

## 1. 推奨アーキテクチャ
- フロントエンド: Next.js（App Router）
- BFF/API: Next.js Route Handler / Server Actions（必要に応じて導入）
- 配備: Vercel

## 2. 開発基盤の前提（Issue #2）
- Next.js App Router を採用し、React Server Components 前提で構成する
- TypeScript を標準とし、型安全を前提に実装する
- スタイリングは Tailwind CSS を使用する
- パッケージ管理は npm を第一候補とする
- import alias は `@/*` を使用する

## 3. ディレクトリ責務方針（初期）
- `src/app`: ルーティング・レイアウト・ページなどUIエントリ
- `src/features`: 機能単位のUI/ロジックを段階的に配置
- `src/lib`: 共通ユーティリティや将来の外部接続境界
- UI層は表示と入力に集中し、予算/価格計算などの業務ロジックは `features` / `lib` 側へ分離する

## 4. 将来導入方針
- Supabase / Auth / DB（マイグレーション含む）は次PR以降で段階的に導入する
- AI補助、通知、栄養管理などはMVP 1以降で追加する

## 5. セキュリティ方針
- 入力バリデーション（Zod想定）
- 機微情報の最小ログ化
- AI API送信データの最小化と同意設計（将来対応）

## 6. エラーハンドリング方針
- ユーザー向け: 行動可能なメッセージ
- 開発者向け: 原因分類（認証/権限/バリデーション/外部依存）
- 未確定: エラーコード体系

## 7. ログ方針
- 操作イベントとエラーを分離
- 食事/価格/健康データ本文はログ出力しない

## 8. 将来の拡張方針
- 家族共有（組織・ロール）
- 管理画面の段階導入
- 外部連携（Appleヘルスケア等）
